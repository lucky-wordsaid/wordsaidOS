Scratch 操作系统定义、分类与 UI 标准（修订版 1.5）

1. 前言
1.1 目的
本标准旨在为在 Scratch 编程环境及其衍生平台上开发的操作系统类项目（以下简称 ScratchOS）提供明确的定义、功能分类、UI 分类与功能要求。标准旨在规范 ScratchOS 生态，提升项目评价的公平性与可比性，促进跨平台协作与技术进步。

1.2 适用范围
本标准适用于所有在 Scratch 或其衍生平台（如 CCW、TurboWarp、PenguinMod、Gandi IDE、02engine、MetaCrea、Tier、Mind+、编程猫 等）上开发并自称“操作系统”的项目。发布形式（.sb3、.html、.exe 等）不影响其适用性。

1.3 核心原则
功能导向：评价应聚焦于项目的功能性实现、用户体验与创意表现。

技术透明：项目需明确声明其技术依赖与实现方式，确保评审环境公开、公平。

社区共建：鼓励开源共享、跨平台协作与社区交流，推动标准持续完善。

2. 术语与定义
ScratchOS：满足本标准第4章最低功能要求的 Scratch 项目。

Scratch UI：未达到完整 ScratchOS 功能标准，仅提供用户界面的项目（详见第5章）。

纯 ScratchOS（Pure-ScrOS）：全部核心逻辑由 Scratch 积木实现。

混合 ScratchOS（Hybrid-ScrOS）：核心功能依赖外部语言（如 C、Python）或自定义扩展实现。

虚拟机 ScratchOS（VM-ScrOS）：在图灵机或虚拟机基础上实现的 ScratchOS。

内存 ScratchOS（In-Memory-ScrOS）：主要在内存中运行，不依赖本地持久化存储。

图形用户界面（GUI）：用户通过图形元素进行交互的界面。

命令行界面（CLI）：用户通过文本命令进行交互的界面。

应用程序（App）：在 ScratchOS 中运行的独立功能模块。

技术依赖透明性：项目需明确声明是否依赖外部代码、插件或转换工具。

跨平台兼容性：能够在多个 Scratch 衍生平台上正常运行的能力。

非线性动画：在运行时速度不一的动画。

3. 技术分类标准
3.1 核心实现分类
3.1.1 纯 ScratchOS（Pure-ScrOS）
定义：全部创造性逻辑与功能均通过 Scratch 积木实现。

允许使用：官方或社区公认扩展（如文本朗读、TurboWarp 扩展）、云变量、浏览器插件（如 JavaScript 插件）。

要求：项目简介中应声明所使用的主要扩展功能。

3.1.2 混合 ScratchOS（Hybrid-ScrOS）
定义：核心功能依赖非 Scratch 语言（如 C、Python）编写的外部代码或自定义扩展。

要求：项目简介中必须标注“Hybrid-ScrOS”，并列出外部依赖与技术实现方式。

3.1.3 虚拟机 ScratchOS（VM-ScrOS）
定义：在图灵机、虚拟机或模拟器基础上构建的 ScratchOS。

要求：需说明虚拟机实现方式及其与操作系统的集成关系。

3.1.4 内存 ScratchOS（In-Memory-ScrOS）
定义：系统核心与应用程序主要在内存中运行，不依赖本地持久化存储。

允许：配合云服务实现数据同步或功能扩展。

3.2 界面形态分类
标准版（GUI）：提供图形用户界面的 ScratchOS。

基础版（CLI）：仅提供命令行界面的 ScratchOS。

混合界面：同时支持 GUI 与 CLI 的系统，需注明主要交互模式。

3.3 技术依赖判定补充
以下情况应归类为 Hybrid-ScrOS 并明确声明：
使用非 Scratch 原生语言编写的自定义扩展或插件；
通过代码转换工具（如 Python 转 Scratch）实现核心功能；
依赖自建服务器、数据库或 API 实现系统级功能（如用户认证、数据同步）。

以下情况仍可归类为 Pure-ScrOS：
仅使用 Scratch 官方扩展或社区公认的轻量级插件；
使用编写 Scratch 所用的语言（JavaScript、WebGL），且未引入核心逻辑依赖。

4. ScratchOS 最低功能要求
一个项目必须同时满足以下所有条件，方可被定义为 ScratchOS。

4.1 系统核心与资源管理
必须实现一个系统核心环境，具备以下能力之一：
持续运行的后台管理环境（适用于多任务系统）；
明确的系统调度机制（适用于单用户顺序执行系统）。

4.2 应用程序抽象与管理
提供至少 2 个 功能完整且互不相同的应用程序。
提供明确的启动、切换及退出机制，应用程序退出后控制权须交还给系统核心。

4.3 统一的用户交互模式
GUI 系统：须提供包含图标、菜单或窗口等至少一种图形元素的桌面或启动器界面。
CLI 系统：须提供命令提示符及一套基础命令集。

4.4 系统服务或数据持久化（至少满足一项）
数据持久化：提供系统级机制，允许应用程序对共享的持久化数据进行读写。
公共服务：提供至少一种系统级服务，如通知、主题管理、多用户切换、硬件状态模拟。
内存运行：适用于内存 ScratchOS，可不依赖本地持久化存储。

5. Scratch UI 定义与分类
5.1 Scratch UI 的定义
未达到第4章最低功能要求的项目，仅提供用户界面，不具备完整的系统功能（如进程管理、文件管理、作业管理），应归类为 Scratch UI。

5.2 UI 分类标准
5.2.1 按交互方式分类
键鼠操作（Mouse & Keyboard）：主要依赖鼠标与键盘操作。
触屏操作（Touch）：主要面向触屏设备设计。
纯键盘/按键操作（Pure-Keyboard）：仅依赖键盘或按键操作。

5.2.2 按渲染技术分类
纯角色（Pure-Char）：完全使用角色构建界面。
纯克隆体（Pure-Clone）：主要使用克隆体构建界面。
纯画笔（Pure-Pen）：完全使用画笔绘制界面，可进一步分为：
原版画笔（OEP）
画笔+（Pen Plus）
Simple3D
HTML 界面：主要使用 HTML 扩展嵌入界面。
多技术混合（Mixed）：结合两种或以上渲染技术。

5.3 UI 项目声明要求
所有 Scratch UI 项目应在简介中明确标注 “Scratch UI”，并说明其交互方式与渲染技术。

6. 常用应用程序功能定义标准
为提升 ScratchOS 生态的通用性和用户体验，本标准对下列常用应用程序（App）提供基础功能定义，以作为开发和评价的参考。ScratchOS 内置的应用程序应努力实现或超越这些标准。

6.1 计算器
核心功能：

具备加（+）、减（-）、乘（*）、除（/）四种基本运算。

具备小数（浮点数）计算能力。

具备清除（C）、清除当前输入（CE）、删除（Del）或等价功能。

提供等号（=）执行计算并显示结果。

进阶功能（鼓励实现）：

实现连续运算。

实现括号以改变运算优先级。

实现幂运算、开方等科学计算功能。

实现历史记录查看功能。

UI/UX 要求：

应有明确的数字（0-9）和运算符输入区域。

应有清晰的显示屏（角色或画笔绘制），能正确显示输入表达式和最终结果。

6.2 文件管理器
核心功能：

提供一个可视化或列表化的视图，用于展示当前可访问的文件/应用程序。

支持对项目/文件进行至少一种操作（如：打开/运行、重命名、删除）。

具备基本的导航能力（如：在“文件夹”或文件类别之间切换）。

进阶功能（鼓励实现）：

支持新建文件/文件夹。

支持文件复制或移动。

支持根据名称、类型或日期排序。

提供图标和列表两种视图模式。

实现简单的文件搜索功能。

UI/UX 要求：

界面应清晰展示文件/应用列表及其关键信息（如名称、图标）。

应提供明确的用户操作引导（如按钮、菜单或命令提示）。

6.3 示例文件
核心功能：

至少包含一个可以被文件管理器列出或通过系统直接访问的预置文件。

该文件应能被系统中至少一个其他应用程序（如文本文档、图片查看器）正确打开和识别。

进阶功能（鼓励实现）：

包含多种类型的示例文件（如文本、图片、配置）。

示例文件能演示系统核心功能或应用间协作。

6.4 文本文档
核心功能：

提供文本输入区域，支持键盘输入字符。

支持基本的光标移动（通过鼠标点击或方向键）。

支持对已输入文本进行编辑，包括删除。

具备保存功能，能将当前文本内容以文件形式写入系统的持久化存储或内存中。

具备打开功能，能读取并加载系统存储中的文本文件。

进阶功能（鼓励实现）：

支持文本格式（如粗体、斜体、下划线）或简单排版。

支持查找和替换。

支持多标签页或多个文档同时编辑。

支持导入/导出为通用文本格式。

UI/UX 要求：

应有清晰的编辑区域与状态显示（如文件名、光标位置）。

应有明确的菜单、工具栏或按钮提供核心操作（新建、打开、保存等）。

6.5 Scratch 浏览器
核心功能：

具备一个地址栏或搜索栏，允许用户输入网址（URL）或搜索词。

能够通过系统网络能力（如云变量、扩展）加载并显示指定网页的文本内容或 Scratch 社区的特定页面结构。

支持基本的页面内链接跳转（在 Scratch 环境允许的范围内）。

进阶功能（鼓励实现）：

实现前进、后退、刷新、主页导航按钮。

实现书签收藏功能。

提供简单的浏览历史记录。

尝试渲染基本的 HTML 结构或图片。

安全与兼容性声明：

项目必须在简介中明确声明其网络功能的限制和对 Scratch 政策的遵守情况。

6.6 Scratch 编程器
核心功能：

提供一个可视化界面，允许用户通过拖放或选择的方式构建 Scratch 积木块。

支持创建变量和列表。

具备对角色（Sprite）的基本管理能力（如新建、命名）。

能够将当前编辑的项目以 .sb3 或兼容格式导出到系统存储中。

能够从系统存储中导入并加载 .sb3 或兼容格式的项目文件。

进阶功能（鼓励实现）：

实现积木的复制、删除和成组操作。

支持编写自定义积木（函数）。

集成简单的语法检查或错误提示。

提供角色/背景的绘图编辑器或资源管理器。

实现项目实时运行/调试功能。

UI/UX 要求：

界面布局应参考主流 Scratch 编辑器，区分积木区、编程区和舞台区。

交互应直观，符合 Scratch 用户的操作习惯。

7. 社区规范与最佳实践
开源鼓励：建议项目以开源形式发布，遵循 Scratch 社区规范。

依赖声明：所有项目必须明确声明技术分类、依赖与兼容平台。

原创性与署名：基于他人项目的再创作，须遵守原许可协议并注明来源。

社区参与：鼓励开发者参与社区交流，共同完善标准与生态。

跨平台声明：建议注明兼容的 Scratch 衍生平台。

8. 发布与修订
本标准修订版 1.5 于 2025 年 12 月 27 日发布。
